# Domain Invariants

## Invariantes visuales
1. El sistema debe respetar los colores base oficiales.
2. Fraunces se usa para títulos y momentos de identidad.
3. Inter se usa para contenido funcional, navegación y lectura.
4. El tono visual debe mantenerse cálido, limpio, sobrio y artesanal.
5. El diseño debe conservar consistencia entre desktop y mobile.

## Invariantes de producto
1. La landing debe comunicar claramente quién es la marca y qué ofrece.
2. La navegación principal no debe distraer del CTA principal.
3. La experiencia debe priorizar claridad antes que decoración.
4. El contenido no debe depender de backend para el MVP si no está definido.
5. No se debe introducir complejidad funcional fuera del alcance del MVP.

## Invariantes de implementación
1. No se implementa un feature sin spec aprobado.
2. Toda decisión estructural debe ser trazable a un documento o ADR.
3. Cada feature debe tener criterios de aceptación verificables.
4. No se agregan dependencias sin justificación documentada.
