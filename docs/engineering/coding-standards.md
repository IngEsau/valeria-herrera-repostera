# Coding Standards

## Regla madre
No se escribe código sin spec suficientemente sólido y aprobado.

## Principios
- Mantener componentes pequeños y explícitos
- Separar contenido, estilos y comportamiento con criterio
- Evitar lógica implícita y nombres ambiguos
- Priorizar legibilidad sobre cleverness
- Mantener bajo acoplamiento entre secciones

## Convenciones
- Nombres descriptivos
- Estructura consistente por feature
- Evitar componentes “god component”
- Evitar estilos duplicados sin razón
- Centralizar tokens visuales

## Accesibilidad
- Contraste suficiente
- Jerarquía semántica correcta
- Navegación clara
- CTA visibles y distinguibles
- Responsive validado

## Calidad
- Cada feature debe tener:
  - spec
  - edge cases
  - test plan
  - definición de terminado

## Prohibiciones
- No agregar librerías arbitrarias
- No improvisar contenido funcional no aprobado
- No mezclar cambios estructurales con cambios cosméticos sin documentarlo
